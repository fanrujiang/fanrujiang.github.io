Maven简介

概述

Apache Maven是专门用于管理和构建Java项目的工具. 基于项目对象模型(POM)的概念，通过一小段描述信息来管理项目的构建、报告和文档

官网：http://maven.apache.org/

Maven模型

项目对象模型 (Project Object Model)

依赖管理模型(Dependency)

插件(Plugin)

Maven仓库和坐标

仓库

项目中，引入依赖jar包后，加载顺序，本地仓库 à 远程仓库 à 中央仓库

本地仓库：自己计算机上的一个目录。本地仓库有，则在项目直接引用;

远程仓库(私服)：一般由公司团队搭建的私有仓库。本地仓库没有，则去远程仓库下载到本地仓库(前提是配置了远程仓库);

中央仓库：Maven团队维护的唯一仓库。本地和远程仓库都没有,先从中央仓库下载到远程仓库，再从远程仓库下载本地仓库。

Maven基本使用

Maven常用命令

①compile ：编译 ②clean：清理 ③test：测试 ④package：打包 ⑤install：安装

命令行方式操作，需要进入pom.xml所在的目录。

第一次使用,需要下载相关插件。

Maven生命周期

生命周期:描述构建一次项目过程经历了多少个事件。Maven 对项目构建的生命周期划分为3套

clean(清理项目)：清理产生的target目录

default(构建项目)：例如编译，测试，打包，安装等,x详细参考《default生命周期详解.png 》

Site(生成项目站点)：产生报告，发布站点等

同一生命周期内，执行后边的命令，前边的所有命令会自动执行

IDEA使用Maven

IDEA配置Maven

选择 IDEA中 File –> Settings

搜索 maven

设置 IDEA 使用本地安装的 Maven，并修改配置文件路径

创建Maven项目

创建模块，选择Maven，点击Next

填写模块名称，坐标信息，点击finish，创建完成

编写 HelloWorld，并运行

导入Maven项目

选择右侧Maven面板，点击 + 号

选中对应项目的pom.xml文件，双击即可

如果没有Maven面板，选择 View à Appearance à Tool Window Bars

操作Maven项目

双击即可执行操作

依赖管理

导入jar包的坐标组成

dependencies:依赖引入坐标,负责依赖引入管理

dependency: 引入可选依赖坐标,负责引入可使用的依赖

groupId，artifactId，version:依赖的项目信息

点击刷新按钮，使坐标生效

坐标参考:https://mvnrepository.com/

导入jar包坐标快捷方式

在 pom.xml 中 按 alt + insert，选择 Dependency

在弹出的面板中搜索对应坐标，然后双击选中对应坐标

点击刷新按钮，使坐标生效

设置坐标自动生效

选择右侧Maven操作界面，

选择auto-reload-mode中的第一个选项。

依赖范围

通过设置坐标的依赖范围(scope)，可以设置 对应jar包的作用范围：编译环境、测试环境、运行环境。

范围

依赖范围 编译classpath 测试classpath 运行classpath 例子
compile(默认值) Y Y Y logback
test - Y - Junit
provided Y Y - servlet-api
runtime - Y Y jdbc驱动
system Y Y - 存储在本地的jar包

默认值：compile
